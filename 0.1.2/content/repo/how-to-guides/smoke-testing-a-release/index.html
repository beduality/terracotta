    Smoke Testing a Release¶

    Use this checklist after a release to confirm the artifacts, registry state, and public sites are all correct end-to-end.

    You can run most of these checks automatically with:

    uv run python scripts/smoke_test_release.py 0.1.2

    Add --build-from-tag, --gradle-e2e, or --sdk-e2e to include the optional end-to-end tests (these require a local Gradle installation).

    Before you start¶

    • You will need the release version, e.g. 0.1.1.
    • Have gh, curl, gpg, jar/unzip, JDK 21, and a working Gradle project ready.
    • Optional but recommended: a throwaway test project for Gradle plugin and SDK checks.

    1. GitHub release¶

    • [ ] The release exists at https://github.com/beduality/terracotta/releases/tag/v0.1.1.
    • [ ] The tag is v0.1.1 and points to the release commit. 
      gh release view v0.1.1 --json tagName,targetCommitish,url
    • [ ] The release title is Release v0.1.1.
    • [ ] The release body matches the ## [0.1.1] section from CHANGELOG.md.
    • [ ] The JAR assets are attached:
    • terracotta-core-0.1.1.jar
    • terracotta-gradle-plugin-0.1.1.jar
    • terracotta-provider-modrinth-0.1.1.jar
    • [ ] No -sources or -javadoc JARs are missing from Maven Central if expected.

    2. Maven Central artifacts¶

    Check that all three modules are live with the right version and metadata.

    VERSION=0.1.1
GROUP=io.github.beduality
ARTIFACTS=(
  terracotta-core
  terracotta-gradle-plugin
  terracotta-provider-modrinth
)

for a in "${ARTIFACTS[@]}"; do
  curl -sI "https://repo1.maven.org/maven2/${GROUP//.//}/${a}/${VERSION}/${a}-${VERSION}.pom"
done

# Gradle plugin marker
MARKER_GROUP=io.github.beduality.terracotta
MARKER_ID=io.github.beduality.terracotta.gradle.plugin
curl -sI "https://repo1.maven.org/maven2/${MARKER_GROUP//.//}/${MARKER_ID}/${VERSION}/${MARKER_ID}-${VERSION}.pom"

    For each artifact verify:

    • [ ] .pom is downloadable and the version is 0.1.1.
    • [ ] .jar is downloadable.
    • [ ] -sources.jar is downloadable.
    • [ ] -javadoc.jar is downloadable.
    • [ ] .pom.asc signature is present and the GPG key is the expected signing key.
    • [ ] .jar.asc signature is present.
    • [ ] .module Gradle metadata is published (for the Gradle plugin).

    The 0.1.1 release was validated as 5 Maven Central components:

    terracotta-provider-modrinth io.github.beduality library

    3. Source tree at the release tag¶

    • [ ] The v0.1.1 tag exists in the repository. 
      git fetch --tags origin
git checkout v0.1.1
    • [ ] gradle.properties contains version = 0.1.1.
    • [ ] pyproject.toml contains version = "0.1.1".
    • [ ] CHANGELOG.md has a ## [0.1.1] - YYYY-MM-DD header and an empty ## [Unreleased] section.
    • [ ] README.md plugin sample uses version 0.1.1.
    • [ ] The build passes from the tagged tree. 
      ./gradlew spotlessCheck build

    4. Artifact integrity¶

    Download the released JARs and verify their contents.

    mkdir -p /tmp/terracotta-smoke && cd /tmp/terracotta-smoke
VERSION=0.1.1
for m in terracotta-core terracotta-gradle-plugin terracotta-provider-modrinth; do
  curl -O "https://repo1.maven.org/maven2/io/github/beduality/${m}/${VERSION}/${m}-${VERSION}.jar"
  jar tf "${m}-${VERSION}.jar" | head -n 20
  jar tf "${m}-${VERSION}.jar" | wc -l
done
    • [ ] Each JAR is a valid ZIP with class files.
    • [ ] No obviously missing packages or META-INF issues.
    • [ ] The Gradle plugin JAR contains the plugin descriptor under META-INF/gradle-plugins/.
    • [ ] The provider JAR references the core classes.

    5. Gradle plugin end-to-end¶

    Create a minimal project that applies the plugin from Maven Central and runs the dry-run task.

    mkdir -p /tmp/terracotta-plugin-smoke && cd /tmp/terracotta-plugin-smoke
cat > settings.gradle.kts <<'EOF'
rootProject.name = "terracotta-plugin-smoke"
EOF

cat > build.gradle.kts <<'EOF'
plugins {
    id("io.github.beduality.terracotta") version "0.1.1"
}

terracotta {
    artifactFile.set(layout.projectDirectory.file("dummy-artifact.jar"))
    providers {
        create("modrinth") {
            projectId.set("dummy-project-id")
        }
    }
    name.set("Smoke Test")
    summary.set("Smoke test project")
    description.set("Testing the 0.1.1 release.")
    license.set("MIT")
    tags.set(listOf("test"))
    gameVersions.set(listOf("1.20.1"))
    loaders.set(listOf(io.github.beduality.terracotta.core.model.TerracottaLoader.PAPER))
    environment.set(io.github.beduality.terracotta.core.model.TerracottaEnvironment.SERVER_ONLY)
}
EOF

# The plugin needs an artifact file to plan with; create a placeholder.
touch dummy-artifact.jar

    Run:

    ./gradlew terracottaPlan --stacktrace
    • [ ] The plugin resolves from Maven Central without manual repository configuration.
    • [ ] terracottaPlan completes successfully (or fails only due to missing Modrinth token, not class/plugin errors).
    • [ ] terracottaApply can be invoked in --dry-run or token-less mode without crashes.

    6. SDK / library end-to-end¶

    Create a small Kotlin/Java project that resolves the core and provider artifacts from Maven Central.

    mkdir -p /tmp/terracotta-sdk-smoke && cd /tmp/terracotta-sdk-smoke
cat > build.gradle.kts <<'EOF'
plugins {
    kotlin("jvm") version "2.3.21"
}

repositories {
    mavenCentral()
}

dependencies {
    implementation("io.github.beduality:terracotta-core:0.1.1")
    implementation("io.github.beduality:terracotta-provider-modrinth:0.1.1")
}
EOF

cat > src/main/kotlin/Smoke.kt <<'EOF'
import io.github.beduality.terracotta.core.model.TerracottaLoader
import io.github.beduality.terracotta.core.model.TerracottaEnvironment

fun main() {
    val loader = TerracottaLoader.PAPER
    val env = TerracottaEnvironment.SERVER_ONLY
    println("Resolved $loader in environment $env")
}
EOF

    Run:

    ./gradlew build --stacktrace
    • [ ] Dependencies resolve from Maven Central.
    • [ ] The project compiles.

    7. Documentation¶

    • [ ] The docs site loads at https://beduality.github.io/terracotta/.
    • [ ] The version selector or banner shows 0.1.1.
    • [ ] The Gradle plugin installation snippet uses version "0.1.1".
    • [ ] The SDK installation page references 0.1.1.
    • [ ] The API reference links for core, gradle-plugin, and provider-modrinth load without 404s.
    • [ ] No broken internal links reported by the docs build.

    8. Changelog and commit integrity¶

    • [ ] The release commit on main matches the tag. 
      git log --oneline v0.1.1 -1
    • [ ] The commit updated gradle.properties, pyproject.toml, uv.lock, and CHANGELOG.md. 
      git show --stat v0.1.1
    • [ ] No unexpected files were modified in the release commit.

    9. Rollback / incident readiness¶

    If any check fails:

    • [ ] You know whether Maven Central published successfully (irreversible) or failed before tagging.
    • [ ] If the GitHub release is wrong but Maven Central is correct, you can edit the release or re-tag manually.
    • [ ] If the tag points to the wrong commit, run uv run scripts/release.py rollback locally and re-trigger the workflow after fixing the issue.

    10. Final sign-off¶

    • [ ] GitHub release page looks correct.
    • [ ] Maven Central has all three library artifacts, the Gradle plugin marker, sources, javadoc, and signatures.
    • [ ] Tagged source tree builds and version strings match.
    • [ ] Gradle plugin resolves and runs in a clean project.
    • [ ] SDK resolves and compiles in a clean project.
    • [ ] Documentation site reflects the new version and has no broken links.

    If all boxes are checked, the 0.1.1 release is fully verified.
