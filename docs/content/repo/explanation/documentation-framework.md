# Documentation Framework

Terracotta docs follow the [Diátaxis](https://diataxis.fr/) framework, organized by user intent:

| Type | Question it answers | Example |
|------|---------------------|---------|
| **Tutorial** | How do I learn this? | [Implement a Custom Provider](../../modules/core/tutorials/implementing-a-custom-provider.md) |
| **How-to Guide** | How do I do X? | [Adding Modrinth to the Gradle Plugin](../../integration/how-to-guides/adding-modrinth-to-gradle-plugin.md) |
| **Reference** | What are the exact details? | [Config Schema](../../modules/core/reference/config-schema.md) |
| **Explanation** | Why does it work this way? | [Core Architecture](../../modules/core/explanation/architecture.md) |

Each page belongs to exactly one type. Mixing types in a single page makes it harder for readers to find what they need.
